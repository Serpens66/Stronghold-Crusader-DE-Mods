Lua Scripting
Asset Mods may optionally include a Scripts/init.lua entry point. When present, the Script Extender loads it into an isolated Lua table environment on every map session: including new games, save loads, and Map Editor opens.
This is the same sandboxed environment system used by Lord AI scripts: your globals live in a table named after your mod's GUID, so they never leak into or from other scripts.
Lifecycle Entry Points



Function
When called




mod_init()
Fresh map start


mod_load(false)
Save game loaded


mod_load(true)
Map opened in Map Editor


mod_unload()
Just before the map unloads



All four functions are optional. If one is absent the Script Extender logs a debug message and continues.
Minimal Example
-- BepInEx/plugins/MyMod/Scripts/init.lua

function mod_init()
    print("MyMod: map started!")
end

function mod_load(isEditor)
    print("MyMod: loaded (editor=" .. tostring(isEditor) .. ")")
end

function mod_unload()
    print("MyMod: unloading")
end
Asset Replacement Guide
Replacing Textures
To replace a vanilla texture, place your replacement at the exact same relative path.
Example: Replace Main Menu Background
Vanilla Path: Assets/GUI/Images/MenusBackground.png
Your File: YourMod/Override/Assets/GUI/Images/MenusBackground.png
BepInEx/plugins/YourMod/
  Override/
    Assets/
      GUI/
        Images/
          MenusBackground.png  # Your replacement
No code required. The game will automatically use your file instead of the vanilla asset.
Adding New Textures
New textures follow the same pattern, but you choose the path.
Example: Add a Custom Icon
BepInEx/plugins/YourMod/
  Override/
    Assets/
      GUI/
        MyMod/
          sword-icon.png
Reference in XAML or code:
<Image Source="/Assets/GUI/MyMod/sword-icon.png" />
Relative Paths: When referenced from XAML, paths are relative to the XAML file's location.

Replacing Unit Sprites
Unit sprites use packed texture atlases with optional team color masks.
Finding Sprite Names
You first need to extract them the AssetBundles via tools such as the AssetRipper (See further down below)
Use the Sprite Previewer Tool to:

View all unit sprites in the game
Export specific sprites
Generate team color masks

Replacement Process

Export the vanilla sprite using the tool (e.g., body_archer-169.png)
Create your replacement (must match original dimensions)
Create a team color mask named body_archer-169_m.png
Place both files in Override/Sprites/

BepInEx/plugins/YourMod/
  Override/
    Sprites/
      body_archer-169.png      # Your custom archer sprite
      body_archer-169_m.png    # Team color mask

Team Color Masks
The _m.png mask defines which pixels are affected by team colors

Note: Detailed mask documentation is not available. Use the Sprite Previewer to export reference masks from vanilla sprites.


Replacing Unit Sprite Atlases

NOTE: This is an experimental feature.

For replacing an entire unit's sprite set at once, use an Atlas Override. Instead of replacing sprites individually, you provide a single texture atlas containing all frames.
Folder Structure
Place your atlas under Override/Atlas/<gmFileName>/:
BepInEx/plugins/YourMod/
  Override/
    Atlas/
      body_bedouin_healer/
        atlas.png        # Your full replacement atlas texture
        atlas_m.png      # Team color mask (optional)
        atlas.json       # Frame definitions
The folder name must exactly match the game's internal GM file name (e.g. body_bedouin_healer, anim_windmill, tile_castle). The system auto-discovers and applies it at load time.
atlas.json Format
Two schemas are supported:
Simple:
{
  "pixelsPerUnit": 64,
  "frames": [
    {
      "name": "body_bedouin_healer-0",
      "rect": { "x": 0, "y": 0, "w": 44, "h": 97 },
      "pivot": { "x": 0.386, "y": 0.196 }
    },
    {
      "name": "body_bedouin_healer-1",
      "rect": { "x": 44, "y": 0, "w": 44, "h": 97 },
      "pivot": { "x": 0.386, "y": 0.196 }
    }
  ]
}
Game raw format (exported directly from AssetRipper: one file per frame, usable as-is):
{
  "m_Name": "body_bedouin_healer-86",
  "m_Rect": { "m_X": 5183, "m_Y": 7212, "m_Width": 43.9, "m_Height": 96.9 },
  "m_Pivot": { "m_X": 0.386, "m_Y": 0.196 },
  "m_PixelsToUnits": 64
}
Here's the updated note block to add directly under the JSON format examples in the docs:


Field Reference:
rect: Pixel rectangle of the frame within the atlas texture. Origin is bottom-left. x/y is the bottom-left corner of the frame, w/h are its dimensions in pixels.
pivot: Normalised anchor point within the frame (0..1 range, bottom-left origin). { "x": 0.5, "y": 0.0 } = bottom-center. This is the point the game anchors to the unit's world position, so it should sit at the unit's feet.
pixelsPerUnit: How many pixels represent one Unity world unit. The game uses 64 universally. Increase proportionally for higher-resolution sprites to maintain correct world-space size (e.g. 128 for 2× resolution renders).

Partial Overrides
You don't need to include every frame. Any frame omitted from the JSON falls back to the original game sprite automatically.
Supported GM Groups
Atlas overrides work for all sprite groups, not just units:



Folder name

Replacing Sounds and Music
Sound replacement supports format conversion (.ogg can replace .wav).
Sound Effects
Example: Replace Chicken Sound
Vanilla: Sounds/chicflap_01.wav (variants up to _02)
Your File: YourMod/Override/Sounds/chicflap_01.ogg
BepInEx/plugins/YourMod/
  Override/
    Sounds/
      chicflap_01.ogg  # Your replacement (can be .wav or .ogg)
Music Tracks
Example: Replace Arabic Music Track
Vanilla: Sounds/Music/oud_03.wav
Your File: YourMod/Override/Sounds/Music/oud_03.ogg
BepInEx/plugins/YourMod/
  Override/
    Sounds/
      Music/
        oud_03.ogg  # Your replacement track
Format Notes:

The API handles .ogg < - > .wav conversion automatically
Higher quality .ogg files are recommended (smaller file size)

Locale-Specific Assets
The Asset API supports locale-aware asset resolution. By placing assets inside a Locales/<lang>/ subfolder within your Override/ directory, you can ship language-specific versions of any asset - sounds, textures, XAML files, and so on.
When the game requests an asset, the system checks in this order:

Override/Locales/<current-language>/... (exact language match, e.g. de-DE)
Override/Locales/en/... (English fallback, skipped if current language is already en-US)
Override/... (global / language-agnostic fallback)

This applies transparently to all asset types -  no extra code is needed.
Folder Structure
BepInEx/plugins/YourMod/
  Override/
    Locales/
      en-US/
        Sounds/
          AI/
            greeting.ogg        # English voice line
      de-DE/
        Sounds/
          AI/
            greeting.ogg        # German voice line
      fr-FR/
        Sounds/
          AI/
            greeting.ogg        # French voice line
    Sounds/
      AI/
        greeting.ogg            # Fallback for all other languages
Example: Locale-Specific Sounds
To replace a sound differently per language, simply place your files in the matching locale folder and let the system resolve them automatically:
Override/
  Locales/
    en-US/
      Sounds/
        chicflap_01.ogg    # English replacement
    de-DE/
      Sounds/
        chicflap_01.ogg    # German replacement
  Sounds/
    chicflap_01.ogg        # Fallback for any other language
If a player is running the game in Italian and no it-IT locale folder exists, they receive the global fallback.

NOTE: The locale key format is BCP 47 (e.g. en-US, de-DE, fr-FR). The English fallback folder uses en-US. Placing assets directly in Override/ (without a Locales/ subfolder) continues to work as before and acts as a universal fallback.


Working with AssetBundles (BepInEx Mods Only)
AssetBundles allow you to load complex Unity assets like 3D models, materials/shaders and prefabs.
Creating AssetBundles

Use Unity Editor (matching game's Unity version)
Build your assets as an AssetBundle
Export the .bundle file

Loading AssetBundles
using SHCDESE.API;

// Load bundle (cached automatically)
AssetBundle bundle = GameBundleManagerAPI.Instance.LoadBundle(
    "AssetBundles/MyMod_bundle", 
    unloadOnMapChange: true  // Free memory when map closes
);

// Load an asset from the bundle
GameObject prefab = bundle.LoadAsset<GameObject>("MyCustomPrefab");
Material material = bundle.LoadAsset<Material>("MyCustomMaterial");

// Instantiate in game
GameObject instance = Instantiate(prefab);
Shader Handling
Unity's shader stripping can cause issues. Use this helper for materials:
Material mat = GameBundleManagerAPI.Instance.LoadMaterialWithShader(
    "AssetBundles/MyMod_bundle",
    "MyMaterial",
    "MyShader"
);

Localization and Translation System
The Script Extender provides a translation system for multi-language support and runtime text overrides.
Quick Overview
Localization Files: Place crusader.txt files in your mod's Locales/ directory:
BepInEx/plugins/YourMod/
  Locales/
    en-US/crusader.txt    # English
    de-DE/crusader.txt    # German
    fr-FR/crusader.txt    # French
Files are automatically loaded when your mod is registered. The format matches the game's native crusader.txt structure.
Runtime Overrides (Lua):
-- Permanent (cleared on map unload)
Translate_SetLookUpText("TEXT_KEY", "Custom Text")

-- Temporary (persist until released)
local handle = Translate_SetTemporaryLookUpText("TEXT_KEY", "Persistent Text")
Translate_ReleaseTemporaryOverride(handle)
Priority: Temporary overrides -> Permanent overrides -> Localized files -> Original game text
See Translation API Guide for detailed documentation, crusader.txt format specification, and C# API usage.
